Lista Encadeada

Após compreendermos o funcionamento dos Tipos Básicos e as Structs, vamos pôr em prática a visualização da memória para algumas estruturas básicas de dados como a lista encadeada.

Para tal, como estamos tratando de informações em 32 bits, tanto o tipo ponteiro quanto o tipo inteiro ocuparão 4 bytes, desta forma podemos inferir que já estarão devidamente alinhados, portanto cada elemento ocupará 4 bytes em sequência, desta forma:

n1:
    Endereço : Bytes
    0xff972584 : 02 00 00 00 00 00 00 00
    //          |  inteiro  | end. NULL |
n2:
    Endereço : Bytes
    0xff932380 : 01 00 00 00 84 25 97 ff
    //          |  inteiro  |  end. n1  |
