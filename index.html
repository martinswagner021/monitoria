Apresentação do projeto

Este projeto tem como objetivo demonstrar a estrutura de armazenamento de dados na memória na linguagem C ensinada nas aulas de Programação de Computadores II.

Aqui veremos alguns códigos exemplares, seus endereços de memória e valores armazenados.

#include <stdio.h>

int main(){
    int x = 5;
}

No exemplo acima, vemos a variável x recebendo valor 5 e portanto obtemos na memória algo similar à:

x:
    Endereço : Int
    0xffde32bc : 05 00 00 00

Na saída do código temos o endereço em hexadecimal correspondente à variável x e o valor de seus 4 bytes armazenados, também em hexadecimal. Lembrando que uma variável do tipo int possui 32 bits, ou o equivalente aos 4 bytes apresentados.

Além disso, é importante ressaltar a ordem em que os dados estão sendo dispostos, isso ocorre por conta da abordagem utilizada pelo sistema para armazenar informações na memória, esse conceito chama-se Endianness, no exemplo o sistema está utilizando o formato Little Endian para armazenar os bytes, neste método os bytes menos significativos são armazenados no início do endereço, desta forma os dados aparentam estar na ordem contrária ao que estamos acostumados a ler, pois os bits menos significativos estão sendo lidos primeiro.

Por exemplo, para o dado em hexadecimal 0x12345678 armazenado no endereço fictício 0x1000, estaria disposto em Little Endian desta forma:

Endereço : Dados
0x1000   : 78
0x1001   : 56
0x1002   : 34
0x1003   : 12

E por este motivo, temos o valor 05 00 00 00 sendo apresentado,ao invés do esperado 00 00 00 05.

Concluímos a apresentação de como as informações e exemplos estarão dispostos para tratar com mais detalhes a relação de bytes e cada tipo de variável em C na próxima seção Tipos Básicos.
